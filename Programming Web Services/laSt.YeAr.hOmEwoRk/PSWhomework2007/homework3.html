Homework 3


Aims of this exercise

The aims of this exercise are:

To understand the data structure of UDDI
To learn to publish and find UDDI business information using Java
To learn to setup a UDDI registry

Introduction

  • Download uddi4j from http://uddi4j.sourceforge.net/. The current version is 2.0.5. Put uddi4j.jar  to your classpath.
  • Download soap4j from http://ws.apache.org/soap/.  The current version is 2.3.1. Put soap.jar  to your classpath.
  • Download javamail from http://java.sun.com/products/javamail/. The current version is 1.4. Put mail.jar to your classpath.
  • Download JavaBeans Activation Framework from  http://java.sun.com/products/javabeans/glasgow/jaf.html. The current version is 1.1. Put activation.jar to your classpath.


Exercise

1.      Read the FindBusinessEntity.java and PublishBusinessEntity.java  example in the textbook (pp327 -- pp334). The code can be downloaded from
http://www.imit.kth.se/courses/2g1523/Homeworks/FindBusinessEntity.java
http://www.imit.kth.se/courses/2g1523/Homeworks/PublishBusinessEntity.java. 

2.      Setup a private UDDI registry as described here: “http://www.devx.com/Java/Article/21390/”. A summary of that article is as follow:

To setup your own UDDI registry, you need to install the following tools. But if you already have installed all or some of them you don’t need to do it again. Please note that the links provided as suggestions and you may use other tools with same functionality (for example you may use another database management system instead of MySQL).

Step 1 of 5: Install these tools:

Java 2 SDK (http://java.sun.com/j2se/index.jsp)
J2EE (http://java.sun.com/javaee/index.jsp)
Web server (http://tomcat.apache.org/index.html)
SOAP (http://ws.apache.org/axis/)
Database (http://www.mysql.com/)
UDDI (http://ws.apache.org/juddi/)

After installing them, you need to proceed with these steps:

Step 2 of 5: Setup the web application:

The registry works as a web application, therefore you need to copy the UDDI web aplication directory to your application server directory. For instance you may copy “C:\juddi-0.9rc4\webapp\juddi” to “C:\Program Files\Apache Software Foundation\Tomcat 5.5\webapps\juddi”.

Step 3 of 5: Setup the database:

After installing your database management system, you need to create UDDI databases and configure the parameters. There are SQL scripts which do it automatically. You can find them in “C:\juddi-0.9rc4\sql\”. Choose the right script for your database system. For example the script for MySQL is in “C:\juddi-0.9rc4\sql\mysql” with name “create_database.sql”.

Step 4 of 5: Create publisher information:

In order to be able to publish, the publisher needs a username and password in the database. To insert this information there is another SQL script. For example for MySQL it is “insert_publishers.sql” in “C:\juddi-0.9rc4\sql\mysql”. You can modify this script, enter your information and then run it to insert publisher information into the database.

Step 5 of 5: Start the web server:

After starting the web server (or restarting it if it was running before copying UDDI web application folder) the registry is ready to use. You can test it by opening the address “http://<host>:<port>/juddi” in a browser. If you use default settings for Tomcat this address will be “http://localhost:8080/juddi”.

3.      Compile PublishBusinessEntity.java with javac PublishBusinessEntity.java. Execute it with java PublishBusinessEntity username password. The username and password are what you created for publisher in previous step. Remember, you have to delete the 30th line, java.security.Security.addProvider(new com.ibm.jsse.JSSEProvider());, otherwise you will get error.

4.      Publish the service you built in Exercise 2. 

5.      Compile FindBusinessEntity.java with javac FindBusinessEntity.java. Execute it with java FindBusinessEntity. Find the service you published in step 4.

6.      Based on FindBusinessEntity.java, write a  code to locate all business entities that have name "SkatesTown".



Deliverable

You are required to deliver a report by the given deadline.

The report must include:

  • a short description of your activities for installation of the above-mentioned tools (what problems did you face)
  • a protocol of compilation and execution of FindBusinessEntity.java
  • a source code for location of all business entities that have name "SkatesTown" and results of running this code
  • results of your registration with UDDI registry (you don't have to disclose your password in the report)
  • results of compilation of PublishBusinessEntity.java with your ID and password and results of publishing you CalculatorService from Exercise 2 in the UDDI registry


The report should be submitted by email in doc, pdf or ps format both to Course coordinator and to Teaching assitant. As for all exercises you are expected to do it in group of 2 persons.
