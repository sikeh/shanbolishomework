Aims of this exercise

The aims of this exercise are:
    To learn the basic data structure of WSDL and SOAP files.
    To learn how to publish a piece of Java code as a Web service using axis.
    To learn how to invoke a Web service by SOAP message using Java.
Introduction


This is a guide to using Axis server to enable Web service creation,
deployment and invocation. The server can be used in a variety of platforms, for example, Windows and Linux. In this guide, we give instruction based on Windows XP.

This guide assumes the readers have already basic knowledge of Windows
XP and Java. Otherwise, the readers should visit the toolkits' home
pages where more detail user manual can be found.

Tomcat

Tomcat is the servlet container that is used in the official reference
implementation for the Java Servlet and JavaServer Pages technologies.
The Java Servlet and JavaServer Pages specifications are developed by
Sun under the Java Community Process. Tomcat can be used as a platform
to support the deployment and invocation of web services developed in
Java language. Tomcat distribution can be downloaded from
http://tomcat.apache.org/index.html.

Before the installation, make sure the correct JDK version has already been installed. Also make sure that an environment variable JAVA_HOME has been set to the path of the directory into which you installed the JDK 1.4.2 release. Here, we suggest the readers create a file setenv.bat under c:\webserv directory and all needed environment variables are set after execute setenv.bat. A sample setenv.bat is shown as below (Please note that you need to change the version numbers to your installation version numbers):

@set JAVA_HOME=c:\cygwin\usr\local\j2sdk1.4.1_01
@set CATALINA_HOME=c:\webserv\jakarta-tomcat-5.0.24
@set AXIS_HOME=c:\webserv\axis-1_3
@set BPWS4J_HOME=c:\webserv\BPWS4j-2.0

@set CLASSPATH_BPWS4J=%CATALINA_HOME%\common\lib\mail.jar
@set CLASSPATH_BPWS4J=%CLASSPATH_BPWS4J%;%CATALINA_HOME%\common\lib\activation.jar

@set CLASSPATH_AXIS=%CATALINA_HOME%\webapps\axis\WEB-INF\lib\axis.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\commons-discovery.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\commons-logging.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\jaxrpc.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\log4j-1.2.4.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\saaj.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\axis-ant.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\wsdl4j.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\webapps\axis\WEB-INF\lib\xercesImpl.jar
@set CLASSPATH_AXIS=%CLASSPATH_AXIS%;%CATALINA_HOME%\common\lib\servlet.jar

@set CLASSPATH=%CLASSPATH%;%CLASSPATH_BPWS4J%;%CLASSPATH_AXIS%;%AXIS_HOME%\;%AXIS_HOME%\samples\

Copy the downloaded file into the c:\webserv and unpack it. Set an variable CATALINA_HOME to point to the tomcat directory.

Start the server by %CATALINA_BASE%\bin\startup.bat
Stop the server by %CATALINA_BASE%\bin\shutdown.sh

If you see the following message, then the installation is successfull:

Using CATALINA_BASE:   c:\webserv\jakarta-tomcat-5.0.24
Using CATALINA_HOME:   c:\webserv\jakarta-tomcat-5.0.24
Using CATALINA_TMPDIR: c:\webserv\jakarta-tomcat-5.0.24\temp
Using JAVA_HOME:       c:\cygwin\usr\local\j2sdk1.4.1_01


You will also see a new console. DO NOT close it when the server is running. The console will close automatically after you shutdown the Tomcat server.

You can then test Tomcat server in browser. Start a web browser and visit http://localhost:8080. If you see the start page of the Apache Jakarta Project then  the Tomcat is running correctly.


Axis

Axis is an Apache implementation of the SOAP  submission to web services. Axis distribution can be downloaded from http://ws.apache.org/axis/. The current reliable version is 1.4.

Unpack the downloaded file. Use an environment variable AXIS_HOME to point the unpacked directory.

Copy the axis directory from %AXIS_HOME%\webapps into the %CATALINA_HOME%\webapps and then restart Tomcat.

Now you can test whether AXIS is working properly. Go to http://localhost:8080/axis, you will see the administration page for Axis. Click Validateand you will see the Axis Happiness Page. If it reports any component missing you can find the link given in that page and download the library files to %CATALINA_HOME%\common\lib directory. After that don't forget to restart Tomcat and to validate axis again. 


Exercise
  • We have a calculator Web service with four operations. These operations can calculate addition, subtraction, multiplication and division. A WSDL document for this Web service can be found here
  • Generate the service using WSDL2Java. You should use -s parameter to generate wsdd file for deploy.
  • Complete the code in Impl class. Compile the generated Java files. 
  •  Deploy the generated service in axis. Remember to copy the class  files to %CATALINA_HOME%\webapps\axis\WEB-INF\classes directory.
  • Construct both incoming and outcoming SOAP messages for invoking the "add" operation 
  • Write a  Java program to invoke this Web service. The functions of this Java program are as follows: 
    •  the user should be able to choose an  operation for calculation. 
    •  the user is required to input the value of two input parameters. The result is returned to the user after the invocation of the correct operation of the Web service.

Deliverable

The report must include:

  • a short description of your activities for installation of the above-mentioned tools (what problems did you face)
  • a protocol of compilation of the WSDL service to Java
  • Completed code for the implementation of the service
  • Protocol of deployment of the service in the axis.
  • Texts of incoming and outcoming  SOAP messages for invoking "add" operation
  • Source code of Java program for invocation of the Calculator service according to required functionality in the Exercise.

The report should be submitted by email in doc, pdf or ps format both to Course coordinator and to Teaching assitant. As for all exercises you are expected to do it in group of 2 persons.
